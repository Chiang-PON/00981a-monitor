4. 維持嚴謹的「雙軌主題 (Light/Dark)」與「無 Emoji」的高質感終端機 UI 設計。

## 🛠️ 第一部分：後端爬蟲引擎實作 (`main.py`)

### 1. 擴充收錄清單與分軌推播機制
* **資料庫全收錄 (`CRAWL_LIST`)**：需爬取以下 18 檔標的存入歷史 CSV：
  `0050` (大盤), `00980A` ~ `00995A` (全台 16 檔主動式 ETF), `009816` (自訂專屬)。
* **LINE 專屬推播 (`LINE_NOTIFY_LIST`)**：為了不干擾使用者，LINE Notify **僅可推播**以下 7 檔：
  `0050`, `00980A`, `00981A`, `00982A`, `00984A`, `00985A`, `009816`。

### 2. 導入「真實資金」計算基礎 (True Money Flow)
* 新增 `fetch_tw_stock_prices()` 函數，於爬蟲啟動時同步抓取 TWSE(上市) 與 TPEx(上櫃) 的「當日收盤價」。
  - TWSE API: `https://openapi.twse.com.tw/v1/exchangeReport/STOCK_DAY_ALL`
  - TPEx API: `https://www.tpex.org.tw/openapi/v1/tpex_mainboard_quotes`
* 在寫入 CSV 時，新增 `price` 欄位。
* **防呆機制**：程式需向下相容。若讀取舊版 CSV 發現無 `price` 或 `weight` 欄位，應以 `0.0` 處理，不可報錯。
* **Bug Fix 提醒**：確保使用 `argparse` 時語法正確為 `parse_args()`，不可拼錯。

## 🛠️ 第二部分：前端儀表板引擎實作 (`generate_web.py`)

### 1. 真實資金換算與排序
* 讀取 CSV 時，計算 `amount = (shares * price) / 10` (轉換為「萬元」單位)。
* **UI 呈現**：在買賣超張數下方，新增等寬字體顯示投入金額，格式如 `(約 1.5 億)` 或 `(約 5,000 萬)`。
* **排序邏輯**：清單的排序需以「張數差值 (diff)」為主，「投入資金大小 (amount)」為輔。

### 2. 產業板塊輪動分析 (Sector Rotation)
* 在 Python 中內建 `SECTOR_MAP` 字典，將常見台股映射至對應產業 (如 台積電=半導體, 鴻海=電腦週邊 等，未定義者歸為「其他」)。
* **UI 呈現**：在 `GLOBAL_CONSENSUS` (全家族共識) 及各別 ETF 頁面上方，新增「Top 3 資金淨流入/流出板塊卡片」，以紅綠色顯示淨買入或淨賣出之總金額。

### 3. 權重變動率 (Weight Delta)
* 計算 `w_diff = 今日權重 - 昨日權重`。
* **UI 呈現**：在個股名稱旁標示當前權重與變動，例如 `W: 2.5% (+0.5%)` 或 `W: 1.2% (-0.2%)`，並依據正負值套用買賣顏色。

### 4. 雷達搜尋與五日微型圖 (Sparklines)
* **UI 呈現**：在搜尋結果 (Search View) 列表右側，利用 JS 動態讀取過去 5 個交易日的數據，繪製 5 根微型柱狀圖 (Sparklines)。買進為向上紅柱，賣出為向下綠柱，無動作為灰柱。高度按比例計算。

### 5. [ AI_AGENT ] 深度分析發射器
